Giới thiệu về ESP8266, ESP32, C++ và Wokwi

Chào mừng đến với bài thuyết trình thực hành IoT

  • Mục tiêu: Hiểu và áp dụng ESP8266, ESP32, C++
  • Công cụ: Wokwi để mô phỏng

Giới thiệu ESP8266

ESP8266 là một vi điều khiển tích hợp Wi-Fi giá rẻ, được phát triển bởi Espressif Systems, rất phổ biến trong các dự án IoT.

  • Đặc điểm kỹ thuật: CPU 32-bit Tensilica L106, 80 MHz, RAM 64 KB, Flash tối đa 4 MB.
  • Kết nối: Wi-Fi 802.11 b/g/n, hỗ trợ chế độ Station, SoftAP.
  • Ứng dụng thực tế: Cảm biến nhiệt độ qua Wi-Fi, điều khiển đèn thông minh, hệ thống tưới cây tự động.
  • Cách sử dụng cơ bản: Lập trình qua Arduino IDE, kết nối với GPIO để điều khiển thiết bị.

Giới thiệu ESP32

ESP32 là vi điều khiển nâng cấp của ESP8266, được phát triển bởi Espressif Systems, hỗ trợ cả Wi-Fi và Bluetooth, phù hợp cho các dự án IoT phức tạp.

  • Đặc điểm kỹ thuật: CPU lõi kép Xtensa LX6, 240 MHz, RAM 520 KB, Flash tối đa 16 MB.
  • Kết nối: Wi-Fi 802.11 b/g/n, Bluetooth 4.2/BLE, hỗ trợ nhiều giao thức.
  • Ứng dụng thực tế: Nhà thông minh (điều khiển qua ứng dụng), robot điều khiển từ xa, hệ thống giám sát an ninh.
  • Cách sử dụng cơ bản: Lập trình qua Arduino IDE hoặc ESP-IDF, hỗ trợ nhiều GPIO và cảm biến.

So sánh ESP8266 và ESP32

ESP8266 và ESP32 là hai vi điều khiển phổ biến trong IoT, nhưng chúng khác nhau về hiệu năng, tính năng và ứng dụng. Dưới đây là so sánh chi tiết:

Khía cạnh ESP8266 ESP32
CPU 1 lõi Tensilica L106, 80 MHz 2 lõi Xtensa LX6, lên đến 240 MHz
RAM 64 KB 520 KB
Flash Tối đa 4 MB Tối đa 16 MB
Kết nối Wi-Fi 802.11 b/g/n Wi-Fi 802.11 b/g/n + Bluetooth 4.2/BLE
Số GPIO 17 36
Tiêu thụ điện Thấp, ~20 mA (chế độ hoạt động) Hơi cao hơn, ~70 mA (chế độ hoạt động)
Ứng dụng lý tưởng Dự án đơn giản (cảm biến, đèn) Dự án phức tạp (nhà thông minh, robot)
Giá cả Rẻ hơn (~$2-$4) Mắc hơn (~$4-$7)

Lựa chọn giữa ESP8266 và ESP32 phụ thuộc vào yêu cầu dự án: ESP8266 phù hợp với ngân sách thấp, trong khi ESP32 lý tưởng cho hiệu năng cao và đa kết nối.

Giới thiệu C++ trong IoT

C++ là ngôn ngữ lập trình mạnh mẽ, được sử dụng trong lập trình nhúng như Arduino. Nó hỗ trợ các khái niệm cơ bản như biến, hàm, vòng lặp, và điều kiện.

Các khái niệm cơ bản:

  • Kiểu biến: int, float, char, bool (ví dụ: int x = 10; float y = 3.14;).
  • Vòng lặp: for, while (dùng để lặp lại hành động).
  • Điều kiện: if-else (dùng để ra quyết định).
  • Hàm: Tái sử dụng code (ví dụ: void setup(), void loop(), void blinkLed()).

Ví dụ code trong Arduino:

          int ledPin = 13; // Kiểu biến: int
          bool isOn = false; // Kiểu biến: bool
          
          void setup() {
            pinMode(ledPin, OUTPUT); // Khai báo chân
          }
          
          void loop() {
            for (int i = 0; i < 5; i++) { // Vòng lặp for
              digitalWrite(ledPin, HIGH);
              delay(500);
              digitalWrite(ledPin, LOW);
              delay(500);
            }
          
            if (isOn) { // Điều kiện if-else
              digitalWrite(ledPin, HIGH);
            } else {
              digitalWrite(ledPin, LOW);
            }
          
            blinkLed(3); // Gọi hàm
            
          }

          void blinkLed(int times) { // Hàm tự khai báo 
            for (int i = 0; i < times; i++) {
              digitalWrite(ledPin, HIGH);
              delay(1000);
              digitalWrite(ledPin, LOW);
              delay(1000);
            }
          }

Code trên điều khiển LED nhấp nháy 5 lần bằng vòng lặp for, sau đó dùng if-else và gọi hàm tùy chỉnh.

Cài đặt môi trường C++ cho Arduino

Cài đặt PlatformIO trên VSCode:

  1. Cài đặt VSCode: Tải từ code.visualstudio.com.
  2. Cài PlatformIO: Mở Extensions (Ctrl+Shift+X), tìm "PlatformIO IDE", nhấn Install.
  3. Tạo project: Nhấn Ctrl+Alt+N, chọn "PIO Home" > "New Project", đặt tên, chọn ESP8266/ESP32.
  4. Mở project: Chọn thư mục project, chỉnh sửa file main.cpp.

Cài đặt Arduino IDE:

  1. Tải Arduino IDE: Từ arduino.cc.
  2. Cài đặt: Chạy file cài đặt, hoàn tất quá trình.
  3. Thêm ESP8266/ESP32: File > Preferences, thêm URL board manager (ESP8266: "http://arduino.esp8266.com/stable/package_esp8266com_index.json", ESP32: "https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json").
  4. Cài board: Tools > Board > Boards Manager, tìm và cài ESP8266/ESP32.
  5. Tạo project: File > New, viết code, chọn board, nhấn Upload.

Giới thiệu Wokwi

Wokwi là một nền tảng mô phỏng trực tuyến tiên tiến, hỗ trợ lập trình và thử nghiệm các dự án IoT với ESP8266, ESP32, Arduino, STM32, và Raspberry Pi Pico mà không cần phần cứng thực tế.

  • Đặc điểm nổi bật: Chạy trong trình duyệt, hỗ trợ Wi-Fi ảo, tích hợp với VSCode, và cung cấp thư viện linh kiện phong phú.
  • Tính năng: Mô phỏng cảm biến, động cơ, màn hình, và giao tiếp IoT (MQTT, HTTP); hỗ trợ MicroPython và C++.
  • Ứng dụng: Học lập trình nhúng, kiểm tra ý tưởng, và chia sẻ dự án qua liên kết.
  • Hướng dẫn sử dụng: Truy cập wokwi.com, tạo dự án mới, chọn bo mạch (ESP8266), thêm linh kiện, viết code, và chạy mô phỏng.
          void setup() {
            pinMode(LED_BUILTIN, OUTPUT);
          }
          void loop() {
            digitalWrite(LED_BUILTIN, HIGH);
            delay(1000);
            digitalWrite(LED_BUILTIN, LOW);
            delay(1000);
          }

Code trên mô phỏng LED nhấp nháy trên ESP8266 trong Wokwi.

Hướng dẫn sử dụng Wokwi với PlatformIO trên VSCode

Wokwi tích hợp với PlatformIO trong VSCode, giúp mô phỏng các dự án IoT. Cần cả file wokwi.toml và diagram.json để thiết lập mô phỏng hoàn chỉnh.

Các bước thực hiện:

  1. Đảm bảo PlatformIO đã cài: Từ slide trước, bạn đã cài PlatformIO IDE trong VSCode (Extensions > PlatformIO IDE).
  2. Cài extension Wokwi: Trong VSCode, mở Extensions (Ctrl+Shift+X), tìm "Wokwi Simulator", và nhấn Install.
  3. Mở project PlatformIO: Mở thư mục project PlatformIO đã tạo trước đó (hoặc tạo mới: Ctrl+Alt+N > PIO Home > New Project, chọn ESP8266).
  4. Thêm cấu hình Wokwi: Trong thư mục project, tạo file wokwi.toml: 
    [wokwi]
version = 1
firmware = "build/esp8266/esp8266:esp8266:generic/firmware.bin"
elf = "build/esp8266/esp8266:esp8266:generic/firmware.elf"
  5. Tạo sơ đồ mạch: Tạo file diagram.json để định nghĩa mạch (ví dụ: ESP8266 với LED): 
    {
"version": 1,
"author": "Your Name",
"editor": "wokwi",
"parts": [
    { "type": "wokwi-esp8266", "id": "esp", "top": 0, "left": 0 },
    { "type": "wokwi-led", "id": "led1", "top": 50, "left": 100, "attrs": { "color": "red" } }
],
"connections": [
    ["esp:D2", "led1:A", "green", ["v0"]],
    ["led1:C", "esp:GND", "black", ["v0"]]
]
}
  6. Viết code trong PlatformIO: Mở file src/main.cpp, viết code điều khiển LED: 
    #include <\Arduino.h>

void setup() {
    pinMode(D2, OUTPUT); // LED kết nối với chân D2
}
void loop() {
    digitalWrite(D2, HIGH);
    delay(500);
    digitalWrite(D2, LOW);
    delay(500);
}
  7. Chạy mô phỏng Wokwi: Nhấn F1, gõ "Wokwi: Start Simulator", Wokwi yêu cầu có lience, hiện tại sẽ miễn phí nhưng có yêu cầu đăng nhập, hãy bấm chấp nhận.

Lưu ý: Build project PlatformIO trước (PlatformIO: Build). File diagram.json định nghĩa mạch, đảm bảo kết nối đúng với code.

Bài tập về nhà

#include 

const int ledPins[8] = {D2, D3, D4, D5, D6, D7, D8, D9}; // Mảng lưu chân LED
int mode = 0; // Biến chế độ nháy

void setup() {
    for (int i = 0; i < 8; i++) {
    pinMode(ledPins[i], OUTPUT); // Khai báo chân làm output
    }
}

void loop() {
    if (mode == 0) { // Chế độ tuần tự
        for (int i = 0; i < 8; i++) {
            digitalWrite(ledPins[i], HIGH);
            delay(200);
            digitalWrite(ledPins[i], LOW);
        }
    } else if (mode == 1) { // Chế độ đồng loạt
        for (int i = 0; i < 8; i++) {
            digitalWrite(ledPins[i], HIGH);
        }
        delay(500);
        for (int i = 0; i < 8; i++) {
            digitalWrite(ledPins[i], LOW);
        }
        delay(500);
    }
    // Thêm chế độ khác (nháy ngẫu nhiên) nếu muốn
    mode = (mode + 1) % 2; // Chuyển đổi chế độ (0 -> 1 -> 0)
}

Bài tập: Điều khiển 8 bóng LED kết nối với ESP8266/ESP32, lập trình để chúng nháy ở các chế độ khác nhau.

  • Yêu cầu: Sử dụng mảng để lưu chân LED, vòng lặp để điều khiển, và điều kiện để thay đổi chế độ nháy (ví dụ: nháy tuần tự, nháy đồng loạt, nháy ngẫu nhiên).
  • Thời gian: Hoàn thành trong 1 tuần, mô phỏng trên Wokwi hoặc chạy trên phần cứng.
  • Gợi ý: Sử dụng mảng để quản lý các chân LED, vòng lặp for để duyệt, và if-else để chuyển đổi chế độ.

Lưu ý khi lập trình cho người mới bắt đầu

Lập trình cho vi điều khiển như ESP8266/ESP32 có thể khó khăn với người mới. Dưới đây là các lưu ý và lỗi phổ biến để bạn tránh:

  • 1. Kiểm tra chân GPIO: Mỗi board có số lượng và chức năng chân khác nhau. Ví dụ, ESP8266 có 17 GPIO, nhưng một số chân như D0 (GPIO 16) không hỗ trợ ngắt (interrupt). Luôn xem sơ đồ chân (pinout) trước khi kết nối linh kiện, tránh làm hỏng board.
  • 2. Quản lý bộ nhớ: ESP8266 chỉ có 64 KB RAM, dễ bị tràn nếu dùng biến lớn hoặc đệ quy sâu. Tránh khai báo mảng lớn (như int array[1000]) và không dùng String trong vòng lặp (dễ gây lỗi bộ nhớ). Thay bằng char array hoặc tối ưu code.
  • 3. Debug với Serial Monitor: Luôn dùng Serial.begin(115200) trong setup() và Serial.println() để in thông tin debug (như giá trị biến, trạng thái). Nếu không thấy gì trên Serial Monitor, kiểm tra tốc độ baud rate (phải khớp với IDE) hoặc cáp USB.
  • 4. Delay() gây treo chương trình: Dùng delay() (như delay(1000)) sẽ dừng toàn bộ chương trình, ảnh hưởng đến các tác vụ khác (như Wi-Fi). Với người mới, hãy thử dùng millis() để thay thế, ví dụ: if (millis() - lastTime > 1000) để không chặn code.
  • 5. Lỗi kết nối Wi-Fi: Khi dùng ESP8266/ESP32, nếu Wi-Fi không kết nối (WiFi.status() != WL_CONNECTED), kiểm tra SSID/password đúng chưa, hoặc router có bật không. Thêm timeout để tránh treo: if (millis() > 10000) thì thoát.
  • 6. Quên khai báo pinMode: Một lỗi phổ biến là quên pinMode(pin, OUTPUT) trong setup(), dẫn đến chân không hoạt động. Luôn khai báo pinMode() cho các chân điều khiển (INPUT/OUTPUT) trước khi dùng digitalWrite() hay digitalRead().
  • 7. Không kiểm tra giá trị trả về: Các hàm như digitalRead() hay analogRead() có thể trả về giá trị không mong muốn nếu cảm biến lỗi. Luôn kiểm tra giá trị, ví dụ: if (digitalRead(pin) == HIGH) trước khi xử lý, tránh logic sai.
  • 8. Lỗi cú pháp cơ bản: Đừng quên dấu chấm phẩy (;), đóng ngoặc ), hoặc khai báo biến sai. Đọc kỹ thông báo lỗi trong IDE, chúng thường chỉ rõ dòng lỗi.

Lời khuyên: Bắt đầu với dự án nhỏ (như LED nhấp nháy), đọc kỹ tài liệu Arduino, và thử nghiệm trên Wokwi trước khi dùng phần cứng thực.
